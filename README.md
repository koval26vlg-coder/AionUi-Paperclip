# AionPaperclipWorkspace

Эта папка является общей рабочей областью для трёх агентов:

- Codex
- Claude Code
- Gemini CLI

Цель рабочей области: сохранить общий контекст, решения, задачи и результаты работы так, чтобы каждый агент мог понять, что делали другие, и дать оценку их работе.

> Впервые здесь? Начни с **[docs/HOW-TO-USE.md](docs/HOW-TO-USE.md)** — гайд для новичка простым языком.

## Главная идея

Эта папка является общей базой контекста, а не проектом под одну конкретную модель. Если Codex, Claude Code, Gemini CLI или другой агент станет недоступен, следующий агент должен прочитать файлы этой папки и продолжить работу.

## Граница проектов

`D:\AionUi-Paperclip` — это проект общей памяти и координации агентов.

`C:\Users\koval\bat\bitrix24-automation` — отдельный прикладной проект для Bitrix/Bit.Newton аналитики звонков, сделок и CRM.

В этой папке могут храниться журналы, решения и backlog по Bitrix-проекту, потому что агенты работали над ним через общую память. Но код, отчеты и runtime Bitrix-проекта должны оставаться в его собственной папке. Aion Vision (`apps/aion-vision`) показывает состояние SML, а не является интерфейсом Bitrix-аналитики.

## Как пользоваться

1. Открывай эту папку в Codex, Claude Code или Gemini CLI.
2. Перед задачей проси агента прочитать `AGENTS.md`, `docs/START-HERE.md` и `docs/context-packs/context-pack-latest.md`.
3. После задачи агент должен добавить запись в `docs/agent-log/`.
4. Важные решения записываются в `docs/decisions.md`.
5. Текущая картина проекта поддерживается в `docs/current-context.md`.

Подробная инструкция с готовыми фразами находится в `docs/how-to-use-agents.md`.

## Быстрый запуск

Запусти файл:

```text
OPEN-VSCODE-SML.cmd
```

Он откроет рабочую папку в VS Code как общую IDE-оболочку SML. Для агентов используйте `OPEN-CLAUDE-SML.cmd` (Claude Code) или запускайте Codex/Gemini CLI из этой папки.

## Обновление контекстного пакета

Обычно это делает фоновая автоматизация. Вручную команду нужно запускать только если автоматизация отключена:

```powershell
.\tools\build-context-pack.ps1
```

Команда создаст или обновит `docs/context-packs/context-pack-latest.md`. Этот файл можно давать любому новому агенту как быстрый вход в историю работы.

## Автоматическая память

Автоматизация описана в `docs/memory-automation.md`.

После установки автозапуска система сама отслеживает изменения в общей базе и обновляет `docs/context-packs/context-pack-latest.md`.

## Роли агентов

Codex используется как основной инженер: сложный анализ, правки, тесты, ревью, автоматизации и аккуратное изменение файлов.

Claude Code используется как сильный инженерный ревьюер и архитектор: глубокий анализ, второе мнение и выполнение задач.

Gemini CLI используется как независимый ревьюер и аналитик большого контекста.

Cursor, Kiro и MiMo Code выведены из схемы 2026-06-18.

## Общий принцип

Ни один агент не является единственным источником истины. Источником истины являются файлы в `docs/` и правила в `AGENTS.md`.
