# Как пользоваться проектом — гайд для новичка

Этот гайд написан простым языком. Если ты открыл папку впервые — начни отсюда.

---

## 1. Что это вообще такое

`D:\AionUi-Paperclip` — это **общая память для AI-агентов**.

Представь, что у тебя есть несколько помощников (Codex, Claude Code, Gemini CLI). Обычно каждый помнит только свой разговор. Здесь они пишут всё важное в **одну общую копилку** — и любой из них может прочитать, что делали другие, и продолжить работу.

Эта копилка называется **SML (Shared Memory Layer)**. Внутри неё:

- **решения** — что и почему решили;
- **журнал работы** — кто что сделал;
- **факты, задачи, ограничения** — всё, что важно не забыть.

Плюс есть **Aion Vision** — красивая веб-панель, которая показывает состояние этой памяти.

> Главная идея: если один помощник пропадёт (закончилась подписка), другой откроет эту папку, прочитает память и продолжит с того же места.

---

## 2. Что нужно для работы

Всё уже установлено на этой машине. Для справки, проект использует:

- **Python** (в папке `.venv-sml`) — для движка памяти SML;
- **Node.js** — для веб-панели Aion Vision;
- **Ollama** (необязательно) — для «умного» поиска по смыслу. Без него поиск тоже работает, но по словам, а не по смыслу.

Проверить, что движок памяти жив:

```powershell
.\.venv-sml\Scripts\python.exe -m tools.sml.core stats
```

Увидишь сводку: сколько записей, какие агенты, какие типы. Если показал числа — всё в порядке.

---

## 3. Как запустить помощников (агентов)

Активные агенты: **Codex**, **Claude Code**, **Gemini CLI**. Все видят общую память.

| Хочешь запустить | Что сделать |
|------------------|-------------|
| Claude Code | двойной клик на `OPEN-CLAUDE-SML.cmd` |
| Gemini CLI | двойной клик на `OPEN-GEMINI-SML.cmd` |
| Просто открыть папку в редакторе | `OPEN-VSCODE-SML.cmd` |

**Важно:** запускай агента именно из этой папки (через эти `.cmd`), иначе он не увидит общую память.

### Как просить агента работать

Агенты уже знают правила (читают `CLAUDE.md` / `GEMINI.md` / `AGENTS.md` сами). Просто ставь задачу обычными словами на русском. Агент сам:

1. посмотрит в память, что уже делали по теме;
2. выполнит задачу;
3. запишет результат обратно в память.

От тебя ничего лишнего не требуется — «посмотри в память» говорить не надо, это происходит автоматически.

## 4. Как открыть веб-панель Aion Vision

Панель показывает: сколько записей в памяти, кто из агентов активен, граф связей, ленту последних событий, **поиск по памяти** и **здоровье системы**.

**Вариант А — для повседневного использования (рекомендуется):**

```text
двойной клик на START-AION-VISION-SERVE.cmd
```

Он соберёт панель и запустит её на `http://127.0.0.1:8787`. Открой этот адрес в браузере.

**Вариант Б — для разработки панели:**

```text
двойной клик на START-AION-VISION.cmd
```

Запустит панель в режиме разработчика (vite, обычно `http://localhost:5173`).

### Что есть на панели

- **Метрики сверху** — всего записей, актуальная память, агенты, источник.
- **Поиск по памяти** — впиши вопрос, нажми «Найти». Покажет похожие записи с процентом совпадения.
- **Лента SML** — последние события в реальном времени (обновляется каждые 30 секунд).
- **Здоровье системы** — три индикатора:
  - *Наблюдатель* — жив ли фоновый процесс, который обновляет память;
  - *Поиск* — «семантика» (Ollama работает) или «текст» (работает без неё);
  - *Бэкап БД* — когда была последняя резервная копия.
- **Граф связей**, **Пульс системы**, **Активные агенты**, **Типы записей**.

---

## 5. Поиск по памяти

Два способа спросить общую память:

1. **Через панель** — поле «Поиск по памяти», впиши вопрос, нажми «Найти».
2. **Через терминал**:

```powershell
.\.venv-sml\Scripts\python.exe apps\aion-vision\scripts\search-sml.py --query "конверсия за неделю" --limit 5
```

Если рядом с результатом написано `mode=semantic` — поиск шёл по смыслу (Ollama работает). Если `mode=text` — по словам (Ollama выключена, но всё равно ищет).

## 6. Обслуживание (раз в день, по желанию)

### Проверить, что всё живо

```text
двойной клик на STATUS-MEMORY-AUTO.cmd
```

Смотри на строку **Heartbeat**:

- `OK` — фоновый наблюдатель работает, память обновляется;
- `STALE` или `MISSING` — наблюдатель не запущен (память не обновляется автоматически). Это не страшно, но обнови вручную (см. ниже).

### Сделать резервную копию памяти вручную

```powershell
.\.venv-sml\Scripts\python.exe tools\backup-sml.py --verify
```

Создаст проверенную копию в `var\sml\backups\`. Копии нужны на случай, если файл памяти повредится. (Фоновый наблюдатель делает это сам раз в день.)

### Обновить панель вручную (если наблюдатель спит)

```powershell
.\tools\build-context-pack.ps1
```

## 7. Если что-то сломалось

| Симптом | Что делать |
|---------|------------|
| Агент не видит общую память | Запусти его через `OPEN-*-SML.cmd` из этой папки, а не как обычно |
| `stats` ругается / нет чисел | Проверь, что есть файл `var\sml\state.db`; убедись, что не удалена папка `.venv-sml` |
| Панель пустая / «Нет данных» | Запусти `START-AION-VISION-SERVE.cmd`; проверь, что `var\sml\state.db` на месте |
| Поиск пишет «недоступен» | Панель должна быть запущена через `START-AION-VISION-SERVE.cmd` или `START-AION-VISION.cmd` |
| Heartbeat STALE | Фоновый наблюдатель не крутится — это нормально, если машину перезагружали; обнови память вручную |
| Поиск стал «текст» вместо «семантика» | Ollama выключена. Если нужен умный поиск — запусти Ollama. Если нет — оставь как есть, базовый поиск работает |

## 8. Шпаргалка команд

```powershell
# Состояние памяти
.\.venv-sml\Scripts\python.exe -m tools.sml.core stats
.\.venv-sml\Scripts\python.exe -m tools.sml.core ping

# Поиск по памяти
.\.venv-sml\Scripts\python.exe apps\aion-vision\scripts\search-sml.py --query "тема"

# Бэкап с проверкой
.\.venv-sml\Scripts\python.exe tools\backup-sml.py --verify

# Веб-панель (прод-режим)
START-AION-VISION-SERVE.cmd

# Статус автоматизации (heartbeat)
STATUS-MEMORY-AUTO.cmd
```

---

## 9. Чего НЕ делать

- Не удаляй папки `var\sml\` (там вся память), `.venv-sml\` (движок) и `tools\sml\` (код движка).
- Не записывай пароли, токены и ключи в документы или память — движок их блокирует, но лучше не пытаться.
- Не редактируй файлы в `docs\context-packs\` вручную — они пересобираются автоматически.
