## Решения

VS Code считается рабочей оболочкой общей памяти, но не отдельным агентом. Память по-прежнему хранится в SML, `docs/`, context-pack и relationship-map.

## Риски и ограничения

- Команда `code` не доступна в PATH текущей PowerShell-сессии, поэтому запускатели используют прямой путь к `Code.exe`.
- VS Code сам не синхронизирует память; он только дает удобную оболочку для работы с файлами, SML-скриптами и терминалами агентов.

2. В VS Code выполнить `Terminal -> Run Task... -> SML: status memory auto`.
