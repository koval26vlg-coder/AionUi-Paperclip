## Риски и ограничения

- `setx` не меняет переменную в текущей сессии PowerShell. Для существующих процессов потребуется новый сеанс. Для служебных процессов Ollama это несущественно — служба установщика уже слушает только loopback по умолчанию.
- Установка выполнена в user-mode без admin-прав. Служба автозапуска Ollama создана на уровне пользователя — работает только при входе конкретного пользователя. Для инфраструктуры в одного пользователя это нормально.
- Не проверял, включён ли Windows Firewall-профиль на hostname-интерфейс — внешний запрос тайм-аутит, этого достаточно для требования.
