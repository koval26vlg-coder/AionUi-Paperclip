# Добавление автопротокола памяти

## Дата и время

2026-05-10

## Агент

Codex

## Исходный запрос пользователя

Пользователь не хочет каждый раз писать агенту, что нужно искать в памяти. Нужно, чтобы агент сразу знал: если тема есть, он должен сам искать похожее в памяти.

## Контекст перед началом

Уже были настроены файловая память, MCP-сервер `aion-file-memory` и автоматическое обновление контекстного пакета. Но использование памяти еще было описано как команда или рекомендация, которую пользователь мог отдельно просить.

## Что сделано

Добавлен обязательный автопротокол памяти:

- перед любой содержательной задачей агент сам определяет тему;
- выделяет ключевые слова;
- ищет похожее через MCP `aion-file-memory`;
- если MCP недоступен, использует `docs/context-packs/context-pack-latest.md` и поиск по `docs/`;
- учитывает найденный контекст перед ответом или действием.

Правило добавлено в общие правила, Cursor rules и Kiro steering.

## Решения

Поиск в памяти стал поведением по умолчанию, а не ручной просьбой пользователя.

## Риски и ограничения

Агент должен поддерживать чтение правил или MCP. Если конкретная модель игнорирует проектные правила, придется явно дать ей `docs/START-HERE.md` или `docs/context-packs/context-pack-latest.md`.

## Что должен проверить следующий агент

Открыть Cursor или Kiro и задать обычную тему без фразы "посмотри в память". Проверить, использует ли агент MCP или общий контекст автоматически.
