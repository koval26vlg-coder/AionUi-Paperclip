# Spec: bitnewton-typing-and-tests

## Статус
- **Агент:** Gemini CLI
- **Дата:** 2026-05-13
- **Состояние:** Исследование / Планирование

## Цель
Обеспечить типобезопасность и надежность логики оценки (scoring) путем добавления аннотаций типов и создания набора тестов на основе синтетических данных. Это необходимое условие перед глубокой декомпозицией и рефакторингом.

## Требования
1. Добавить аннотации типов во все модули `pipelines/`.
2. Использовать `mypy` для проверки.
3. Создать фикстуры синтетических транскриптов в `tests/fixtures/transcripts/`.
4. Реализовать тесты для функций скоринга:
   - `checklist` (проверка по чек-листу)
   - `objections` (обработка возражений)
   - `discipline` (дисциплина речи)
   - `quality` (качество презентации)

## План реализации

### Этап 1: Исследование и подготовка
1. [ ] Изучить сигнатуры функций в `pipelines/scoring.py`.
2. [ ] Проверить наличие `mypy` и `pytest` в целевом окружении.
3. [ ] Создать структуру папок для фикстур.

### Этап 2: Типизация
1. [ ] Обновить `pipelines/scoring.py` (аннотации).
2. [ ] Обновить `pipelines/kpi.py`.
3. [ ] Обновить `pipelines/runtime.py` и `pipelines/cli.py`.
4. [ ] Запустить `mypy` и устранить ошибки.

### Этап 3: Тестирование
1. [ ] Подготовить 3-5 синтетических транскриптов (хороший звонок, плохой звонок, звонок с возражениями).
2. [ ] Написать тесты, проверяющие извлечение KPI из этих транскриптов.
3. [ ] Убедиться, что `pytest` проходит.

## Риски
- Сложные структуры данных в `scoring.py` могут потребовать создания специальных `TypedDict` или `Pydantic` моделей.
- Текущая логика может содержать скрытые баги, которые всплывут при типизации (это плюс, но потребует времени на фикс).
