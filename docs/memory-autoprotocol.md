# Автопротокол памяти

Этот протокол действует всегда, даже если пользователь отдельно не просит искать в памяти.

## Цель

Агент должен сам проверять, есть ли в общей памяти похожая тема, прошлое решение, задача, ошибка, ограничение или предпочтение.

Пользователь не должен каждый раз писать: "посмотри в память".

## Перед началом любой задачи

Агент обязан:

1. Понять тему запроса.
2. Выделить 3-7 ключевых слов.
3. Найти похожее в памяти.
4. Учесть найденный контекст.
5. Только после этого отвечать, планировать или выполнять задачу.

## Если доступен MCP

Приоритет — новый MCP-сервер `sml` (Shared_Memory_Layer, Python 3.13 + SQLite WAL + LanceDB + Ollama bge-m3). Он обеспечивает семантический поиск на русском, temporal-запросы и стартовый пакет из 6 разделов.

Минимальная последовательность:

1. `sml.startup_pack` — стартовый набор из 6 разделов (природа проекта, решения, активные задачи, предпочтения, ограничения, последние записи журнала).
2. `sml.semantic_query` по теме запроса с лимитом 10–20.
3. `sml.semantic_query` по ключевым сущностям, если тема сложная.
4. При необходимости — `sml.temporal_query` (состояние на конкретную метку времени) или `sml.read` (конкретный Memory_Record по id).

После работы:

1. `sml.add_log`, если была важная работа — создаёт файл в `docs/agent-log/` и Memory_Record типа `agent_log`.
2. `sml.write` с `type = "fact" / "preference" / "constraint" / "timeline_event"`, если появился долгосрочный факт, предпочтение, ограничение или событие хронологии.
3. `sml.add_decision`, если принято архитектурное решение — append в `docs/decisions.md`.
4. `sml.supersede`, если новое решение/факт отменяет предыдущие.
5. `sml.build_context_pack` для пересборки `docs/context-packs/context-pack-latest.md` (автоматически делает и `tools/watch-memory.ps1`, но явный вызов форсирует обновление).

Если `sml` недоступен (таймаут ≥ 5 с или ошибка), используй старый `aion-file-memory` как fallback:

1. `read_context_pack`
2. `search_memory` по теме
3. `add_agent_log` / `add_memory` / `create_handoff` после работы

## Если MCP недоступен

Используй файловый fallback:

1. Прочитай `docs/context-packs/context-pack-latest.md`.
2. Проверь `docs/tasks.md`.
3. Проверь `docs/decisions.md`.
4. Проверь последние записи `docs/agent-log/`.
5. При необходимости используй поиск по файлам в `docs/`.

## Когда можно не искать

Можно не выполнять полный поиск только если пользователь просит:

- простую команду без контекста, например "покажи время";
- чисто техническую проверку одного конкретного файла;
- не использовать память явно.

Даже в этом случае, если запрос связан с агентами, памятью, настройками, проектом, подписками, Cursor, Kiro, Codex или MCP, поиск обязателен.

## Как использовать найденное

Не нужно длинно пересказывать всю память.

Достаточно коротко учитывать найденный контекст в решении:

- "В памяти уже есть решение..."
- "Ранее выбрали такой подход..."
- "Есть ограничение..."
- "Последняя активная задача..."

Если найденный контекст противоречит запросу пользователя, сообщи о конфликте.
