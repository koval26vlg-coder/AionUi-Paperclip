# Текущий контекст

Дата создания: 2026-05-10

## Рабочая схема

Проект переводится на чистую связку:

- Codex
- Cursor
- Kiro

AionUi, Paperclip и Hermes больше не используются как основа текущей архитектуры.

## Цель

Сделать так, чтобы три агента работали в одной общей системе:

- знали запросы и результаты друг друга;
- могли оценивать работу друг друга;
- сохраняли общий контекст;
- фиксировали решения и задачи в документах;
- в будущем использовали общую MCP-память.

## Текущая структура

- `AGENTS.md` - единые правила для всех агентов.
- `.cursor/rules/` - правила Cursor.
- `.kiro/steering/` - steering-документы Kiro.
- `docs/agent-log/` - журнал работы агентов.
- `docs/decisions.md` - журнал решений.
- `docs/tasks.md` - список задач.
- `docs/memory/` - подготовка к общей MCP-памяти.

## Следующий шаг

Подключить общую MCP-память. Рекомендуемый вариант: Graphiti MCP для локальной приватной памяти. Альтернатива: Mem0/OpenMemory MCP для более простой настройки.
