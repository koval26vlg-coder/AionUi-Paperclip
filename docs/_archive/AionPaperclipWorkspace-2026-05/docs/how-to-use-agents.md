# Как пользоваться тремя агентами

## Главное правило

Всегда открывай одну и ту же папку:

```text
C:\Users\koval\Documents\Bitrix24\AionPaperclipWorkspace
```

Если агенты работают в разных папках, они не будут видеть общий контекст.

## Запуск

Можно открыть вручную:

- Cursor: `File > Open Folder`
- Kiro: `File > Open Folder`
- Codex: открыть терминал в этой папке

Или запустить:

```text
OPEN-AGENT-WORKSPACE.cmd
```

## Как давать задачу Codex

Пример:

```text
Прочитай AGENTS.md, docs/current-context.md, docs/tasks.md и последние записи docs/agent-log.
Выполни задачу: ...
После работы обнови docs/agent-log, docs/current-context.md и docs/tasks.md.
```

## Как просить Cursor оценить Codex

Пример:

```text
Прочитай AGENTS.md и последнюю запись Codex в docs/agent-log.
Оцени, насколько хорошо Codex выполнил задачу.
Проверь измененные файлы, найди риски и недостающие проверки.
Не исправляй код, сначала дай ревью.
```

## Как просить Kiro сделать spec

Пример:

```text
Прочитай AGENTS.md, docs/current-context.md, docs/tasks.md, docs/decisions.md и последние записи docs/agent-log.
На основе текущего контекста сделай spec для задачи: ...
Раздели работу на требования, дизайн, задачи, критерии приемки и риски.
```

## Как просить Codex оценить Cursor или Kiro

Пример:

```text
Прочитай журнал работы Cursor/Kiro в docs/agent-log.
Проверь результат как инженерное ревью.
Скажи, что сделано правильно, где риски, что нужно исправить и какие проверки нужны.
```

## Что делать после каждой важной работы

Попроси агента:

```text
Зафиксируй результат в docs/agent-log и обнови общий контекст.
```

Если агент этого не сделал, следующий агент может потерять часть истории.
