# Публикационный пакет Telegram-канала "ИИ в дело"

Пользователь попросил продолжить следующий шаг через workflow: быстро прочитать 20 постов, поправить под личный голос, затем подготовить канал, закреп и перенос лид-магнита в Google Doc/Notion/PDF. Внешне ничего не публиковать и канал не создавать без отдельного подтверждения.

## Исходные артефакты
Предыдущий workflow `2026-06-23-133613-058359-контент-пакет-telegram-канала-ии-в-дело` завершен. В нем есть:
- `posts-20.md`;
- `lead-magnet-10-automations.md`;
- `publishing-guide.md`.

## Цель workflow
Подготовить публикационный пакет:
- voice profile канала;
- 20 постов в едином голосе, ready-to-paste;
- лид-магнит в doc-ready формате;
- channel setup kit: название, описание, закреп, CTA, checklist создания канала и переноса лид-магнита.

## Голос по умолчанию
Если пользователь не дал персональные примеры, принять голос: спокойный практик, короткие фразы, обращение на "вы", без инфобизнеса, без обещаний легкого заработка, с конкретными действиями и шаблонами.

## Ограничения
- Не создавать Telegram-канал.
- Не публиковать посты.
- Не создавать Google Doc/Notion/PDF во внешних системах.
- Не использовать секреты/API-ключи.
- Не запускать долгие фоновые процессы.
- MiMo/Antigravity handoff runtime ранее был нестабилен; fallback фиксировать явно.

## Критерии готовности
- Есть `voice-profile.md`.
- Есть `posts-20-ready-to-paste.md` с 20 постами.
- Есть `lead-magnet-doc-ready.md`.
- Есть `channel-setup-kit.md`.
- Workflow финализирован со state `done`.
