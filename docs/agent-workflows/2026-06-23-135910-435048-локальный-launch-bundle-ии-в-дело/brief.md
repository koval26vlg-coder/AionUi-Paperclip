# Локальный launch bundle для канала "ИИ в дело"

Пользователь сказал "давай дальше" после публикационного пакета. Без отдельного подтверждения нельзя создавать Telegram-канал, Google Doc/Notion/PDF или публиковать наружу. Поэтому следующий безопасный шаг: подготовить локальный launch bundle для ручного старта.

## Исходные артефакты
Предыдущий workflow `2026-06-23-135126-025594-публикационный-пакет-ии-в-дело` завершен. В нем есть:
- `voice-profile.md`
- `posts-20-ready-to-paste.md`
- `lead-magnet-doc-ready.md`
- `channel-setup-kit.md`

## Цель workflow
Создать локальный launch bundle:
- `00-launch-checklist.md` — пошаговый запуск без внешней записи агентом;
- `01-channel-profile.txt` — название, описание, username candidates;
- `02-pinned-post.txt` — готовый закреп;
- `03-lead-magnet-google-doc.md` — готовый текст для Google Doc/Notion;
- `04-day-1-posts.txt` — первые 2 поста для первого дня;
- `05-leads-tracker.csv` — локальный шаблон учета заявок;
- `06-next-confirmation.md` — что именно пользователь должен подтвердить для внешнего запуска.

## Ограничения
- Не создавать Telegram-канал.
- Не создавать Google Doc/Notion/PDF во внешних сервисах.
- Не публиковать посты.
- Не использовать секреты/API-ключи.
- Не запускать долгие фоновые процессы.
- Фиксировать fallback явно, если MiMo/Antigravity не используются.

## Критерии готовности
- Все launch bundle файлы существуют.
- Workflow финализирован `done`.
- Пользователь получает понятный следующий шаг и список внешних действий, требующих подтверждения.
