## Что получилось хорошо

Иерархическая схема реально сработала: каждый следующий уровень читал и уточнял вывод предыдущего, а не перезаписывал задачу по памяти.

Лучший билет не был искажен между уровнями:

- базовые даты остались 2026-07-15 - 2026-07-25;
- цена без багажа осталась 71 903 RUB;
- цена с багажом 10 кг осталась 74 500 RUB;
- маршрутные точки остались `VOG -> SVO -> PVG -> MNL` и обратно `MNL -> PVG -> SVO -> VOG`.

Antigravity CLI фактически выполнил работу, несмотря на пустой stdout. Это подтверждено `events.jsonl`, handoff-файлами и conversation DB.

## Какие есть риски

Workflow-risk: формальный `MiMo AUTO` не был подтвержден как автономный runtime; L1.0 нужно считать тестовым/формальным уровнем.

Automation-risk: `agy --print` без stdout делает автоматизацию хрупкой, пока нет wrapper.

Travel-risk: цена динамическая, а самый дешевый маршрут может оказаться составным и менее безопасным при задержках.

Decision-risk: пользователь может принять цену как гарантированную. Финальный отчет обязан написать, что это цена Aviasales на момент проверки.

## Что нельзя потерять/исказить дальше

Не смешивать "самый дешевый" и "самый удобный": самый дешевый вариант за 71 903 RUB не самый быстрый и не обязательно самый безопасный.

Не терять, что вариант с багажом стоит 74 500 RUB и выглядит разумнее минимального безбагажного.

Не скрывать, что direct API Aviasales без браузерного контекста блокировался 403, а данные получены через UI.

Не утверждать, что пользователь может купить без дополнительной проверки транзита и оплаты.
