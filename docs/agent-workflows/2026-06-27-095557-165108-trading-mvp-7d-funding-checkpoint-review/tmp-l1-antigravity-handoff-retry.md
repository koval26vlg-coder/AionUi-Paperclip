## На чем основан вывод
1. **Непригодность для paper-forward**: Ветка `funding carry` должна быть отклонена для этапа paper-forward. Текущий вывод об отсутствии готового к запуску экономического преимущества (edge) полностью подтверждается.
2. **Первичность и вторичность блокировщиков**: Ограничение по качеству данных (`min_rows_per_cycle = 9` при строгом пороге `20`) является вторичной проблемой. Первичный блокировщик — экономический: при ослабленных критериях проверки качества количество пригодных рынков (`rank_eligible`) оказалось равным 0. Топовые рынки не проходят валидацию по ключевым параметрам прибыли, комиссий и спотовой ликвидности.
3. **Диагностический статус**: Даже в случае устранения пропусков в сборе данных, стратегия не показывает положительного математического ожидания с учетом транзакционных издержек.

## Что требует доработки
1. **Покрытие коллектора (Collector Coverage)**: При необходимости дальнейшего использования этого источника данных требуется исправить падение плотности строк за цикл (`min_rows_per_cycle = 9` вместо требуемых `20`).
2. **Анализ структуры комиссий (Maker/Fee-tier Economics)**: При расчете безубыточности (break-even) необходимо проверить допущения о комиссиях (например, возможность торговли исключительно лимитными ордерами в качестве maker на более выгодных тарифах).
3. **Выбор направления исследования**: В связи с отсутствием явного преимущества на текущей ветке рекомендуется переключиться на альтернативные семейства сигналов (alternative edge family) или официально завершить работу над данной веткой.

## Какие есть риски
1. **Риск ложного отклонения (False Negative)**: Использование завышенных или неактуальных параметров комиссионных издержек в модели оценки, из-за чего потенциально прибыльная maker-стратегия могла быть оценена как убыточная.
2. **Искажение симуляции из-за пропусков**: Низкая частота обновления данных в циклах может скрывать кратковременные всплески ликвидности или проскальзывания, искажая исторический бэктест.
3. **Потери времени на оптимизацию**: Риск продолжения доработки сбора данных (collector coverage fix) для ветки, экономическая неэффективность которой уже доказана.
