## Что было сделано

MiMo AUTO L1.0 выполнил первичный AUTO-проход по задаче перехода от выбранного drift-arena reference render к рабочему прототипу в `apps/aion-vision`.

Первичный вывод:

- выбранный визуальный язык: drift arena с главным активным автомобилем в центре, статусными секторами вокруг и audit panels по краям;
- отредактированный render с drift-легендой и tuned kei cars должен быть reference, но не единственным UI;
- первый рабочий прототип должен быть read-only, чтобы не открыть новый неконтролируемый путь изменения workflow state;
- данные для первого прохода можно брать как fixture workflow snapshot, затем подключить real workflow files/export.

## На чем основан вывод

Вывод основан на запросе пользователя, предыдущем completed workflow `2026-06-20-091104-901407-drift-agent-dashboard-reference-renders`, отредактированном render `drift-arena-tuned-kei-edit.png`, и текущем brief нового workflow.

## Что получилось хорошо

- Пользователь явно выбрал не линейную эстафету, а arena-style render, поэтому прототип должен уважать этот визуальный выбор.
- Замена финального автомобиля на drift-легенду усиливает характер L5/финальной инстанции.
- Tuned kei cars хорошо ложатся на метафору субагентов: маленькие, быстрые, сопровождают главный уровень, но не перетягивают внимание.

## Что требует доработки

- Нужно решить, будет ли MVP встроен в существующий Aion Vision как отдельный экран, или станет отдельным standalone app. L1.0 рекомендует встроить в Aion Vision, чтобы не плодить новый фронтенд.
- Нужно проверить, не сломает ли новая view существующий SML dashboard.
- Нужно добавить code-native state machine: `active`, `next`, `waiting`, `blocked`, `revision`, `done`.
- Нужно явно показать неизвестные лимиты как `unknown`, а не выдумывать remaining/reset.

## Какие есть риски

- Если использовать PNG как весь экран, это будет красивый, но не рабочий прототип.
- Если dashboard получит кнопки `approve/submit/finalize`, можно повторить риск самовольной мутации workflow state.
- Если в UI оставить слишком много дыма/эффектов, readability станет хуже, чем у старого SML dashboard.

## Что нельзя потерять/исказить дальше

- Рабочий прототип должен быть code-native и интерактивным.
- Рендер является reference, а не финальным интерфейсом.
- Drift-легенда и tuned kei cars должны попасть в визуальный язык прототипа.
- Workflow должен оставаться read-only consumer state, пока отдельный write-gate не будет спроектирован.
