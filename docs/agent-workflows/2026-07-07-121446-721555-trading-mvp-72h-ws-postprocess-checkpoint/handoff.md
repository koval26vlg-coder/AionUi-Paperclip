## На чем основан вывод
- Манифест сбора `ws_collect_ws_durable_72h_20260704_000015.json` подтверждает целостность данных: параметры `completed=true`, `final=true`, `stop_condition=duration_sec`, фактическое время сбора 259199.8 сек (при запрошенных 259200 сек), общее количество сегментов равно 24, число незавершенных сегментов равно 0, ошибки отсутствуют (`errors_count=0`).
- Успешный visible resume после очистки диска позволил получить консистентный набор из 24 сегментов. Неудачная попытка с дефолтным `TotalSec=7200` была корректно замещена и не вошла в финальный манифест.
- Обработка 76 млн строк несет существенные риски исчерпания оперативной памяти (RAM) при единовременной загрузке и требует контроля свободного места на диске под распакованные файлы и промежуточные форматы.
- Переход к фазе replay/grid требует предварительного подтверждения качества данных (data quality gate) на этапе постпроцессинга, что делает обязательным предварительный запуск replay-validation в тестовом режиме (`PlanOnly`).

## Что получилось хорошо

## Что требует доработки
- Скрипт защищенного постпроцессинга (guarded ws-postprocess) должен строго использовать перечень файлов из манифеста `ws_collect_ws_durable_72h_20260704_000015.json`, а не применять glob-поиск по директории, чтобы гарантированно проигнорировать архивную папку `seg_020_incomplete`.
- Реализация потоковой (chunk-by-chunk/streaming) обработки данных в постпроцессинге для предотвращения OOM-ошибок при чтении 76M событий.
- Определение конкретных пороговых критериев качества данных (пропуски, пинг, аномалии) для 72-часового интервала перед запуском replay-валидации.

## Какие есть риски
- Риск переполнения диска на этапе постпроцессинга из-за увеличения объема данных при конвертации raw-событий в структурированный формат.
- Риск исчерпания RAM при неоптимальном парсинге JSON-файлов сегментов большого объема.
- Риск случайного импорта устаревших или поврежденных данных из каталога `seg_020_incomplete` в случае ошибок фильтрации путей.
- Риск задержек выполнения из-за неактивного статуса фонового монитора сбора данных.
