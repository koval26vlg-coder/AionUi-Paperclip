## Цель workflow

Довести внешний запуск до безопасного состояния:

1. После ручного входа пользователя в Telegram Web создать канал `ИИ в дело`.
2. Username пробовать в порядке: `iivdelo`, `iivdelo_ai`, `ii_vdelo`, `iivdelo_ru`, `iivdelo_pro`.
3. Настроить описание канала из launch bundle.
4. Не публиковать закреп и посты без отдельного подтверждения точного текста.
5. Не менять доступ к Google Doc по ссылке без отдельного подтверждения.
6. Зафиксировать итог и ссылки в `final-report.md`.

## Текущий блокер

Для продолжения нужен ручной вход пользователя в Telegram Web через QR-код. Пользователь должен написать: `Вошел в Telegram`.

## Критерии готовности

- Канал создан или явно зафиксирован блокер/невозможность создания.
- Google Doc ссылка сохранена.
- Все внешние публикации и access changes имеют отдельное подтверждение или не выполнялись.
- Workflow финализирован только после реального результата или явного решения пользователя остановиться.
