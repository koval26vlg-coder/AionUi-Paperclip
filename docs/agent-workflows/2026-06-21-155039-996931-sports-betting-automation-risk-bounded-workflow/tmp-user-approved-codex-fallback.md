# User-approved Codex fallback

## Факт

После отчета Codex о том, что Antigravity CLI L1.1 не дал валидный handoff, пользователь ответил: `продолжи`.

## Интерпретация

В контексте предыдущего сообщения Codex, где был запрошен approve fallback для продолжения вместо Antigravity L1.1/L2, это принято как разрешение продолжить workflow через Codex fallback для текущей задачи.

## Граница fallback

- Fallback распространяется на L1.1/L2 текущего workflow.
- Это не меняет глобальную model policy.
- В handoff-и явно записывается, что L1.1/L2 authored by Codex fallback, not Antigravity.
- Запрет на hidden real-money auto-betting и обход правил БК/KYC/CAPTCHA/лимитов/anti-bot остается обязательным.
