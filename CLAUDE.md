# Claude Project Instructions

Ты работаешь в общей рабочей области `D:\AionUi-Paperclip`.

Эта папка является инфраструктурой общей памяти и контекста для агентов. На 2026-06-18 активная рабочая связка: Codex + Claude Code + Gemini CLI (Cursor, Kiro и MiMo Code выведены из схемы). Главная цель: все активные агенты должны быть взаимозаменяемыми, видеть общий контекст и продолжать работу друг друга без потери памяти.

## Язык

Всегда отвечай на русском языке.

Исключения допустимы только для кода, команд, путей, API, логов ошибок и официальных названий.

## Источник истины

Не считай текущий чат единственной памятью.

Важно: обычные Claude web/Desktop projects/chats и OpenClaude/Cowork-сессии не являются общей памятью этого проекта. Общая память находится в локальной папке `D:\AionUi-Paperclip`, SML и relationship-map. Если ты не видишь эти файлы и MCP-сервер `sml`, значит ты работаешь вне общего контура памяти.

Главные источники истины:

- `AGENTS.md`
- `docs/START-HERE.md`
- `docs/context-packs/context-pack-latest.md`
- `docs/current-context.md`
- `docs/tasks.md`
- `docs/decisions.md`
- `docs/agent-log/`
- `docs/relationship-maps/graphify-sml-relationship-map.json`
- MCP-сервер `sml`

Если есть противоречие между чатом и файлами, сначала проверь файлы и SML.

## Автопротокол перед задачей

Перед любой содержательной задачей сам ищи общий контекст. Не жди отдельной просьбы пользователя.

Минимальный порядок:

1. Определи тему запроса.
2. Прочитай `AGENTS.md` и, если доступен, `docs/context-packs/context-pack-latest.md`.
3. Если доступен MCP-сервер `sml`, вызови `sml.startup_pack`.
4. Если доступен `sml.semantic_query`, найди похожие записи по теме запроса.
5. Если есть `docs/relationship-maps/graphify-sml-relationship-map.json`, используй relationship-map как быстрый навигационный слой:

```powershell
& "D:\AionUi-Paperclip\.venv-sml\Scripts\python.exe" "D:\AionUi-Paperclip\tools\query-relationship-map.py" "<тема запроса>"
```

6. Учитывай найденное перед ответом или действием.
7. Если SML недоступен, ищи по `docs/context-packs/context-pack-latest.md`, `docs/current-context.md`, `docs/tasks.md`, `docs/decisions.md` и последним файлам `docs/agent-log/`.

Подробный протокол: `docs/memory-autoprotocol.md`.

## Автопротокол после работы

После важной работы сохрани результат:

1. Добавь запись в `docs/agent-log/`.
2. Если принято важное решение, обнови `docs/decisions.md` или используй `sml.add_decision`.
3. Если появился долгосрочный факт, предпочтение или ограничение, обнови соответствующий слой в `docs/memory/` или используй `sml.write`.
4. Если изменились задачи, обнови `docs/tasks.md`.
5. Если изменилась общая картина проекта, обнови `docs/current-context.md`.

В журнале указывай:

- запрос пользователя;
- краткий план;
- что сделано;
- измененные файлы;
- проверки;
- риски;
- следующие шаги.

## Роль Claude Code

Claude Code в этой системе используется как:

- сильный инженерный ревьюер;
- альтернативный агент для сложного анализа;
- помощник по архитектурным решениям;
- резервный исполнитель задач, если Codex или Gemini временно недоступны.

Claude не должен работать изолированно. Перед выводами и правками он должен учитывать SML, context-pack, карту связей и журнал других агентов.

## MCP

Проектный MCP-конфиг находится в `.mcp.json`.

Ожидаемый сервер:

```text
sml
```

Если Claude Code не видит `sml`, проверь:

```powershell
claude mcp list
```

Если сервер не подключился, открой проект именно из папки:

```powershell
cd /d D:\AionUi-Paperclip
claude
```

## Безопасность

Не записывай секреты, API-ключи, токены, пароли и приватные данные в:

- `docs/`
- `AGENTS.md`
- `CLAUDE.md`
- `.mcp.json`
- SML
- сообщения журнала

Если пользователь случайно прислал секрет, не повторяй его и предложи заменить или отозвать.

## Главное правило

Работай так, чтобы Codex, Gemini CLI или другой агент могли открыть эту папку, прочитать SML и документы, и продолжить без потери контекста.
